## 피그마 실사용 여백값 출처

| 값 | 피그마 사용 위치 |
|---|---|
| `8px`  | 미션 아이템 row padding (`py-8`), 아이콘↔텍스트 gap (`gap-8`) |
| `10px` | 카드 섹션 래퍼 vertical padding (`py-10`) |
| `12px` | 내비게이션 바 상단 padding (`pt-12`), 로고 그룹 gap |
| `13px` | 상단 바 아이콘 그룹 gap (`gap-13`) |
| `16px` | 카드 내부 vertical padding (`py-16`), 상단/내비 horizontal padding |
| `20px` | 화면 horizontal padding, 카드 내부 가로 padding |
| `22px` | 카드 내부 요소 top 위치값 (버디 보내기 카드 레이블 위치) |

## 옵션 A — 피그마 실제값 그대로

**규칙:** 피그마에서 확인된 7개 값을 그대로 토큰화. 조정 없음.

| 토큰명 | 값 | 용도 |
|---|---|---|
| `space-8`  | `8px`  | 아이템 row 여백, 아이콘↔텍스트 간격 |
| `space-10` | `10px` | 섹션 래퍼 vertical padding |
| `space-12` | `12px` | 내비게이션 상단 padding, 로고 gap |
| `space-13` | `13px` | 상단바 아이콘 그룹 gap |
| `space-16` | `16px` | 카드 내부 vertical, 화면 edge padding |
| `space-20` | `20px` | 화면/카드 horizontal padding |
| `space-22` | `22px` | 카드 내부 요소 위치 |

**장점:** 피그마 재현 완벽. 토큰이 실제 화면값과 1:1 대응.  
**단점:** 13px, 22px 등 불규칙 값이 섞여 있어 확장 시 일관성 없는 여백 혼입 위험.

## 옵션 B — 4px 그리드 근사화

**규칙:** off-grid 값을 가장 가까운 4px 배수로 반올림.

| 원본값 | 조정값 | 차이 | 영향 위치 |
|---|---|---|---|
| `8px`  | `8px`  | ±0 | — |
| `10px` | `12px` | +2px | 카드 섹션 래퍼 상하 여백이 살짝 넓어짐 |
| `12px` | `12px` | ±0 | — |
| `13px` | `12px` | -1px | 상단바 아이콘 3개 사이 간격이 1px 좁아짐 |
| `16px` | `16px` | ±0 | — |
| `20px` | `20px` | ±0 | — |
| `22px` | `20px` | -2px | 버디 보내기 카드 내부 레이블 위치가 2px 위로 |

조정 후 토큰 (4단계로 압축됨):

| 토큰명 | 값 | 용도 |
|---|---|---|
| `space-2`  | `8px`  | 아이템 row 여백, 아이콘↔텍스트 간격 |
| `space-3`  | `12px` | 섹션 래퍼 vertical, 내비 상단, 로고 gap, 상단바 gap |
| `space-4`  | `16px` | 카드 내부 vertical, 화면 edge |
| `space-5`  | `20px` | 화면/카드 horizontal, 카드 내부 요소 위치 |

**장점:** 4개 값만으로 일관된 그리드 유지. 확장·추가 시 예측 가능.  
**단점:** 실제 피그마와 최대 2px 차이 발생. 특히 `22px → 20px` 조정은 카드 내부 비율에 육안 차이 가능.
