# Working Buddy Design System

---

## 프로젝트 개요

**워킹버디(Working Buddy)** 는 직장 동료 관계 행동 조언 서비스다.
AI 챗봇으로 대화 상황을 리허설하거나, 개인 맞춤형 행동 제안 및 커뮤니케이션 스타일을 분석해주고,
보상 시스템으로 지속적인 학습을 유도하는 기능을 통해 직장인의 커뮤니케이션을 돕는다.

이 레포는 공모전에서 진행한 워킹버디 작업의 디자인 결정을
**AI 워크플로우로 사후 시스템화한 학습 프로젝트**다.
Figma MCP를 통해 피그마 파일을 직접 읽고, AI와의 단계별 대화로
디자인 토큰과 컴포넌트 스펙을 정제하는 과정 전체를 기록했다.

## AI 활용 방식

이 프로젝트의 핵심은 **프롬프트를 어떻게 설계하느냐에 따라 결과물의 품질이 달라진다**는 것을 직접 확인한 과정이다.

### V1 — 전체 위임의 문제점

> *"피그마 파일 보고 디자인 시스템 만들어줘. Material Design 같은 거 참고해서 발전시켜줘."*

AI에게 판단을 전부 위임했을 때 발생한 문제:

- **컬러 46개 생성** — 피그마 실사용은 7개. 상태색(error/success)도 피그마에 없는데 추가됨
- **타이포 13종 생성** — 실사용은 7종. h1·h2·h3 등 근거 없는 위계 포함
- **컴포넌트 오류** — Badge·Input 같은 미존재 컴포넌트 구현, Checkbox·Nav 누락
- **4px 그리드 강제** — 피그마의 10px, 13px, 22px 같은 실제값 표현 불가

### V2 — 단계별 제어로 전환

> *"피그마에서 확인된 이 값들만 써줘. 없는 건 추가하지 마."*

영역을 쪼개고 매 단계마다 조건을 명시하자 결과가 달라졌다:

- 컬러 → 타이포 → 스페이싱 → 레디어스 → 컴포넌트 순서로 분리 진행
- 각 단계마다 피그마 실사용값을 직접 제시, AI의 임의 확장 차단
- 스페이싱에서는 두 가지 옵션을 AI에게 제안받고 직접 선택 (4px 그리드 근사화)

### V3 — 미결 사항 처리

V2 과정에서 발견된 3가지 미결 사항을 결정:

| 미결 | 결정 | 근거 |
|---|---|---|
| Shadow 토큰 미정의 | 피그마 실사용 3개만 토큰화 | 확장 없음 원칙 |
| TopBar Bold(700) | SemiBold(600)으로 근사 | 시스템 일관성 > 예외 1개 |
| Nav padding-bottom 34px | spacing 토큰 외 고정값으로 명시 | 디자인값이 아닌 iOS 환경값 |

### 단계별 토큰 수 변화

| 영역 | V1 | V3 |
|---|---|---|
| 컬러 | 46개 | 17개 |
| 타이포 사이즈 | 11단계 | 6단계 |
| textStyles | 13종 | 7종 |
| 스페이싱 | 16단계 | 4개 |
| 레디어스 | 8단계 | 3개 |
| Shadow | 8개 | 3개 |
| **합계** | **~102개** | **43개** |
